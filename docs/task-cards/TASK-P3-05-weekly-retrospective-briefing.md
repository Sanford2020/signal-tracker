# TASK-P3-05: Weekly Retrospective Briefing

## Goal

Create a weekly retrospective that explains what changed, revived, verified, cooled, or became high-opportunity over the last week.

## Scope

- Add weekly briefing schemas and backend generator.
- Add `GET /api/v1/briefings/weekly`.
- Add UI mode switch on the briefing page.
- Add tests for weekly sections.

## Acceptance

- Weekly briefing includes changed, resurrected, verified/debunked, opportunity, and cooling/noise sections.
- Window is configurable in days.
- Frontend can switch between daily and weekly briefing views.
